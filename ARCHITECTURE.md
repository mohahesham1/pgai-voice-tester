# Architecture

This project is a **patient simulator voice bot** that places outbound calls to the Pretty Good AI assessment line (`+1-805-439-8008`) and converses naturally with their phone agent. The bot is not a scripted IVR; each call uses an LLM-driven patient persona with a scenario-specific goal (scheduling, refills, edge cases). Audio flows through Twilio Media Streams: inbound audio from the Athena agent is buffered with simple voice-activity detection, transcribed with OpenAI Whisper, passed to the patient LLM, then spoken back with OpenAI TTS converted to 8kHz mu-law for Twilio playback.

Design choices prioritize **real conversational quality** and **submission deliverables**. Twilio handles telephony, dual-channel recording, and webhook delivery; FastAPI hosts the media WebSocket loop and status callbacks. After each call completes, the system saves transcripts, downloads the MP3 recording, and runs a separate LLM QA pass to append structured bug findings to `bug_report.md`. Scenario definitions live in one module so evaluators can see intentional edge-case coverage (weekend scheduling, topic changes, vague requests). Secrets stay in `.env`; only `.env.example` is committed.
